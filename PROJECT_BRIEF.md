# FitCoach AI — Project Brief & Architecture Guidelines

## What Is This Project
Prashant Agarwal wants to build an AI-powered personal fitness coaching web/mobile app. The idea emerged organically from his own experience using Claude as a personal fitness and diet coach over the past week. The core insight is that Claude's contextual, conversational coaching — knowing his history, injuries, diet restrictions, and progress — is vastly superior to any existing fitness app on the market.
The app will be called (working title) FitCoach AI and will be built by Prashant along with his team of 2 developers and 2 interns at Paper Theory Networks.

## The Problem Being Solved
Existing fitness apps fall into two categories:
* Dumb trackers (MyFitnessPal, HealthifyMe) — log food and workouts but give no intelligent guidance
* Generic AI chatbots — no structure, no continuity, no accountability

The gap is a structured tracking experience combined with Claude's contextual intelligence. No app today understands that jowar roti is better than white rice, or that a user with plantar fasciitis cannot do jumping jacks, or that protein has been consistently 40g short for 6 days and breakfast is the root cause.

## Target User
Primary target: 30-45 year old Indian urban professional returning to fitness after a gap. Characteristics:
* Works long hours, sedentary lifestyle
* Eats home-cooked Indian food primarily
* Needs structure and accountability, not just a log
* Cannot afford or sustain a personal trainer
* Understands apps but wants simplicity

## Core Value Proposition
The app should feel like texting your personal coach who knows everything about you — your injury history, what you ate yesterday, that you lifted 10kg last week so should try 10.5kg today, that you skipped gym on Day 4 and had vodka. No other app does this today. This is the moat.

## Key Features Decided

### Must Have (V1 MVP)
1. **Onboarding Flow (One Time)**
   * Basic stats: age, gender, height, current weight, target weight
   * Fitness background: beginner or returning, last active period, equipment access, days per week, preferred time
   * Goals: primary goal (fat loss / muscle / fitness & energy), specific target date or event
   * Diet profile: vegetarian / eggetarian / non-veg, food restrictions (multiselect), supplements taken
   * Health & injuries: multiselect injuries (knee, back, shoulder, foot/PF), medical conditions
   * Summary confirmation screen
   * AI plan generation screen with loading state
2. **Daily Workout Tab**
   * Today's AI-generated workout plan displayed as a table
   * Each exercise row shows: exercise name and planned sets × reps × weight. Photo/image for exercise identification.
   * Editable column for actual reps and weight achieved per set
   * Checkbox to mark each exercise complete
   * Add custom exercise button (with selection assistance based on targeted muscles).
   * Reasons to add this exercise (Claude generated) - why this exercise, target muscle, benefits.
   * Notes for user field per exercise
   * Post-workout: session rating (1-10), how did you feel (Easy/Good/Hard/Exhausted)
   * Claude generates feedback after session is marked complete
   * Tomorrow's plan preview shown after completion
   * Workout history: calendar view with gym days marked, tap to see full detail, PRs highlighted
3. **Diet / Nutrition Tab**
   * Food photo upload as primary input
   * Claude Vision API analyses photo and returns itemised nutrition estimate
   * User can correct any item (name or quantity) before confirming
   * Running daily calorie and protein totals with progress bars
   * Macro breakdown (carbs, fat)
   * Water intake tracker (manual, glasses)
   * Daily coach tip generated by Claude based on current day's numbers
   * Meal timing cards: Breakfast / Lunch / Snack / Dinner
4. **Coach Chat Tab**
   * Direct conversational interface with Claude
   * Full user context injected into every message
   * User can ask anything (modify tomorrow's plan, meal suggestions, injury adjustments)
   * Key differentiator from all other apps
5. **Progress Tab (Basic in V1)**
   * Weekly Monday weight check-in
   * Weight trend line graph
   * Weekly gym sessions bar chart
   * Weekly avg protein bar chart
   * Milestone cards / badges

## Architecture Decisions

### The Core Problem — Claude Has No Memory
Every Claude API call is stateless. All context must be supplied in the prompt every single time. 
**Smart Context Builder — Key Architecture Insight**
Never send all historical data to Claude. Send a structured, compressed context payload (~1,600-2,000 tokens):
1. USER PROFILE (~200 tokens) — always sent
2. AGGREGATED STATS (~300 tokens) — always sent
3. LAST WEEK WORKOUT SUMMARY (~400 tokens) — always sent
4. LAST 3 DAYS DIET LOG (~500 tokens) — always sent
5. TODAY'S CONTEXT (~200 tokens) — always sent
6. USER'S MESSAGE

### Two Separate Data Pipelines
1. **Pipeline 1 — Daily Planning (uses raw DB)**
   * Injects last 2-3 days raw data into prompt. Fast, cheap, always fresh.
2. **Pipeline 2 — Weekly Summary Compression (background job)**
   * Runs every Sunday night via node-cron. Calls Claude API once to compress week into ~300 token summary.
   * Stored in WeeklySummaries collection. Replaces raw daily data in long-term context.

### When NOT to Call Claude API
* **Yes:** Generate workout plan, Analyse food photo, Weekly progress summary, Coach chat response.
* **No:** Log reps/sets to DB, Show calorie total, Progress graphs, Historical lookup.

## MongoDB Collections
* `users` — profile, goals, dietary restrictions, injuries, targets
* `workouts` — daily planned + actual workout documents
* `dietLogs` — daily meal entries with photo URLs and nutrition
* `weeklySummaries` — compressed weekly summaries for context
* `metrics` — weekly weight, measurements, progress photos
* `notifications` — user notification preferences and history

## The Analysis Feature — Critical Product Requirement
Pattern analysis MUST be built into the app.
**Diet Patterns:**
* Breakfast protein by day (table showing egg vs no-egg days)
* "On 4 out of 6 days you skipped eggs at breakfast..."
* Running weekly average vs target
* Weakest meals for protein
**Workout Patterns:**
* Exercises skipped repeatedly flagged
* Progressive overload suggestions
* Sessions missed with context
* Energy level trends
*Appears:* After food photo submission, after workout completion, every Monday, and proactive coach nudges.
